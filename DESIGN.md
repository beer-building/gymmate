# Design System: GymMate

## 1. Overview

**Creative North Star: «Терминал тренировок»**

GymMate выглядит как точный прибор, а не как фитнес-журнал. Интерфейс — табло спортивного оборудования: почти чёрные поверхности, моноширинные подписи в верхнем регистре, вольтовый лайм как индикация действия и прогресса. Цифры (веса, повторы, подходы) — герой каждого экрана; всё остальное — приборная панель вокруг них. Тонкое зерно поверх фона и пружинящий отклик компонентов дают физическое, тактильное ощущение — снаряд в руке, а не страница в браузере.

Система явно отвергает глянец фитнес-индустрии: никаких сток-фото атлетов, мотивационных баннеров, бейджей и конфетти. И равно отвергает безликость корпоративного SaaS: ни серых сеток одинаковых карточек, ни hero-метрик. Это личный инструмент дисциплины — короткий, честный, по делу.

Весь UI измеряется в rem от единой базы (16px десктоп / 18px мобильный): одна точка масштабирования делает приложение крупнее на телефоне, как нативное. Тема следует системной настройке: тёмная — основная (зал, приглушённый свет), светлая — дневной аналог с апельсиновым акцентом на тёплом креме.

**Key Characteristics:**

- Тёмная тема по умолчанию; вольтовый лайм (#c9f231) — единственный голос бренда
- Моноширинные подписи-лейблы в верхнем регистре с разрядкой — язык приборной панели
- Тональная глубина без теней в покое; тень и glow — только ответ на состояние
- Пружинящий тактильный отклик (spring-кривая, сжатие при нажатии)
- Зерно плёнки поверх фона (opacity 0.05) — фактура вместо декорации
- Мобильный в зале — первый экран: нижний таб-бар, safe-area, крупные тач-таргеты

## 2. Colors

Сдержанная палитра: десять нейтральных тонов держат сцену, один акцент бьёт точно.

### Primary

- **Volt, «вольтовый лайм»** (#c9f231, dark) / **«жжёный апельсин»** (#b33d0b, light): энергия разряда. Маркирует первичные действия (заливка кнопок), текущее состояние (активная вкладка, выбранный чип), прогресс и фокус полей. Никогда — декорация. На вольтовой заливке текст всегда цвета фона. Светлое значение подобрано под контраст ≥4.5:1 на всех трёх поверхностях — volt работает как цвет текста мелким кеглем.
- **Volt-dim** (#97b522 / #9a3412): приглушённый шаг акцента для hover-границ и второстепенной индикации, где полный volt кричал бы.
- **Danger** (#ff5d45 / #c81e1e): функциональный (не брендовый) цвет ошибок и разрушительных действий. Кнопки удаления — прозрачные с danger-текстом, заливка не используется.

### Neutral

- **Ink** (#ececdf / #1b1710): основной текст. Тёплый костяной белый на тёмном — не стерильный #fff.
- **Muted** (#8e9095 / #6b6456): вторичный текст, неактивные лейблы, подписи, плейсхолдеры (без прозрачности — opacity роняет контраст).
- **Bg** (#0e0f11 / #f4f0e7): тело страницы; поверх — radial-градиент volt-свечения (6%) у верхней кромки.
- **Bg-raised** (#16181b / #fbf8f1): приподнятые поверхности — карточки-плиты, бегунок табов.
- **Bg-sunken** (#0a0b0c / #e9e4d8): утопленные поверхности — поля ввода, трек табов.
- **Line** (#26282c / #dcd5c6) и **Line-strong** (#3a3d42 / #bdb29c): границы; strong — для интерактивных элементов (поля, чипы), обычная — для разделителей и карточек.

### Named Rules

**Правило одного вольта.** Volt занимает ≤10% любого экрана. Его редкость — его сила: если акцент повсюду, индикация мертва. Прогресс, действие, текущее состояние — больше ничего.

**Правило двух тем.** Тема следует `prefers-color-scheme`, переключателя нет. Каждый новый цвет обязан существовать в обеих темах и проходить контраст ≥4.5:1 для текста.

**Правило относительных оттенков.** Полупрозрачные вариации строятся через `oklch(from var(--token) l c h / α)`, а не новыми hex-значениями — палитра остаётся десятью токенами.

## 3. Typography

**Display Font:** Unbounded (sans-serif fallback)
**Body Font:** Manrope (sans-serif fallback)
**Label/Mono Font:** JetBrains Mono (monospace fallback)

**Character:** Контраст по оси «широкий дисплейный против технического моно»: Unbounded даёт заголовкам вес железной плиты, JetBrains Mono превращает подписи и цифры в показания прибора, Manrope спокойно несёт текст между ними.

### Hierarchy

- **Display** (800, h1–h3, line-height 1.05, letter-spacing -0.01em): только заголовки страниц и секций. `text-wrap: balance` обязателен.
- **Body** (400–600, 0.9375rem–1rem, line-height 1.6): описания упражнений, заметки, текст форм.
- **Label** (600, 0.6875–0.75rem, letter-spacing 0.12–0.22em, UPPERCASE, mono): кнопки, навигация, лейблы полей, чипы, eyebrow-подписи.
- **Числовые данные** (mono): веса, повторы, даты — всегда JetBrains Mono, включая `input[type="number"]` и `input[type="date"]`.

### Named Rules

**Правило табло.** Каждая цифра, которую пользователь сравнивает между подходами или неделями, набирается моноширинным — выравнивание разрядов и есть читаемость прогресса.

**Правило прибора.** Display-шрифт запрещён в UI-элементах: ни кнопок, ни лейблов, ни данных в Unbounded. Единственное исключение — ghost-инпут размера lg, редактирующий заголовок инлайн (он и есть заголовок).

## 4. Elevation

Плоско в покое. Глубину дают три тональных слоя — утопленный (bg-sunken), тело (bg), приподнятый (bg-raised) — и однопиксельные границы между ними. Тени и свечение появляются только как ответ на состояние: hover карточки-плиты, бегунок активной вкладки, подсветка под акцентными элементами. Поверхность, отбрасывающая тень просто так, в этой системе лжёт.

### Shadow Vocabulary

- **Plate hover glow** (`box-shadow: 0 10px 30px -16px oklch(from var(--volt) l c h / 0.45), 0 0 0 1px oklch(from var(--volt) l c h / 0.15)` + `transform: translateY(-2px)`): отклик интерактивной карточки на наведение.
- **Tab thumb** (`box-shadow: 0 2px 8px -2px rgb(0 0 0 / 0.35)`): бегунок табов — единственный элемент с постоянной тенью, потому что он физически «лежит» на треке.
- **Header blur** (`backdrop-filter: blur(14px) saturate(140%)` на полупрозрачном bg): шапка и мобильный таб-бар — функциональное стекло поверх прокрутки, не декоративный глассморфизм.

### Named Rules

**Правило «плоско в покое».** Тень = состояние. Если элемент не интерактивен и не активен — у него нет тени.

## 5. Components

Тактильные и упругие: каждый интерактивный элемент пружинит при нажатии (`transform: scale(0.96)` со spring-кривой `--spring-transition`) и отвечает цветом за 0.15s.

### Buttons

- **Shape:** мягкое скругление (`--radius-sm`, 6px), mono-шрифт, UPPERCASE, `white-space: nowrap`.
- **Primary:** заливка volt, текст цвета фона; на hover — инверсия (прозрачный фон, volt-текст и граница).
- **Ghost:** прозрачный, граница line-strong, текст ink; hover — volt-граница и volt-текст.
- **Danger:** прозрачный, граница line-strong, текст danger; hover — danger-граница. Заливки нет даже на hover.
- **Icon / Icon-filled:** квадрат 2rem без подписи; цвет через `--icon-color`; filled держит постоянную 10%-заливку своего цвета.
- **Sizes:** sm / md / lg; высота md (42px) согласована с инпутами и селектами.
- **Disabled:** opacity 0.4, `pointer-events: none`.

### Chips

- **Style:** прозрачный, граница line-strong, mono UPPERCASE, цвет muted.
- **State:** hover — volt-dim-граница и ink-текст; активный — полная volt-заливка с текстом цвета фона. Используются для фильтров каталога.

### Cards / Containers («плита», `.plate`)

- **Corner Style:** `--radius-lg` (12px).
- **Background:** bg-raised, граница line.
- **Shadow Strategy:** плоско в покое; на hover интерактивной плиты — volt-glow, подъём на 2px и radial-подсветка, следящая за курсором (`--mx`/`--my`).
- **Internal Padding:** 1–1.5rem по содержимому.
- **Заряженная плита (`.plate.volt`):** ключевой блок страницы (форма записи подхода, следующая тренировка) — volt-рамка на 45% прозрачности + мягкая radial-заливка из левого верхнего угла. Side-stripe (`border-left > 1px`) запрещён.

### Inputs / Fields

- **Boxed (default):** фон bg-sunken, граница line-strong, радиус 6px; фокус — volt-граница. Числовые и date-поля — mono.
- **Ghost:** без рамки, пунктирная нижняя граница; фокус — сплошная volt-линия. Размер lg — инлайн-редактирование заголовков display-шрифтом.
- **Label:** mono UPPERCASE 0.6875rem с разрядкой 0.15em, цвет muted (паттерн `.field`).
- **Error:** текст danger 0.875rem под полем.

### Navigation

- **Desktop:** sticky-шапка с blur, mono-ссылки UPPERCASE; текущий раздел — volt-текст (без подложки). Нижняя кромка — однопиксельный volt-градиент.
- **Mobile (≤720px):** фиксированный нижний таб-бар с blur и safe-area; иконка над подписью, активный таб — volt; прячется при фокусе в поле ввода. Профиль живёт в таб-баре.

### Tabs (signature)

Трек bg-sunken с границей; активную вкладку отмечает физический бегунок bg-raised с тенью, который пружинит между позициями (spring по left/width). Вкладки mono UPPERCASE; активная — volt-текст. Вариант sm — pill-форма.

### Difficulty Bar (signature)

Пять сегментов-пилюль; заполненные показывают свой срез общего градиента «зелёный → жёлтый → красный» (#7ed957 → #e8c33b → #e03c31) — шкала нагрузки как индикатор на приборе. Единственное место с градиентом в системе.

## 6. Do's and Don'ts

### Do:

- **Do** маркируй вольтом только действие, текущее состояние и прогресс — ≤10% экрана (Правило одного вольта).
- **Do** набирай сравниваемые цифры (веса, повторы) в JetBrains Mono — выравнивание разрядов читается как график.
- **Do** давай каждому интерактивному элементу пружинящее сжатие при нажатии (`scale(0.96)`, `--spring-transition`) и цветовой отклик за 0.15s.
- **Do** строй полупрозрачности через `oklch(from var(--token) l c h / α)` — без новых hex.
- **Do** проверяй каждый новый цвет в обеих темах и на контраст ≥4.5:1.
- **Do** держи всё в rem: масштабирование на мобильном происходит одной сменой базового font-size.
- **Do** добивай зону нажатия мелких контролов до ≥44px на touch (`@media (pointer: coarse)`: невидимый `::after` у кнопок/чипов/табов, утилита `.hit-target` для текстовых кнопок) — визуальная плотность интерфейса не меняется.
- **Do** добавляй `@media (prefers-reduced-motion: reduce)`-альтернативу каждой анимации — это требование PRODUCT.md.

### Don't:

- **Don't** «глянцевые фитнес-аппы»: сток-фото атлетов, градиентные баннеры, мотивационные слоганы — запрещены (анти-референс PRODUCT.md).
- **Don't** «корпоративный SaaS-дашборд»: серые сетки одинаковых карточек, hero-метрики (большая цифра + мелкий лейбл + градиент), Inter повсюду (анти-референс PRODUCT.md).
- **Don't** «геймификация»: бейджи, стрики, конфетти, маскоты — честные данные вместо игровых механик (анти-референс PRODUCT.md).
- **Don't** «медицинский трекер»: стерильный белый фон, синие акценты, тон надзора (анти-референс PRODUCT.md).
- **Don't** display-шрифт (Unbounded) в кнопках, лейблах и данных — только заголовки (Правило прибора).
- **Don't** постоянные тени на карточках и панелях — тень только как ответ на состояние (Правило «плоско в покое»).
- **Don't** цветные side-stripe границы (`border-left > 1px`), gradient text (`background-clip: text`), декоративный глассморфизм.
- **Don't** модалка как первая мысль: запись подхода и правки — инлайн; модалка — последний выход.
- **Don't** новые градиенты: единственный легальный — шкала сложности difficulty-bar.

Тест на провал: если экран можно принять за дашборд аналитики или баннер фитнес-клуба — он не GymMate. GymMate выглядит как прибор, на котором записывают подход за время отдыха.
